Required Local-Dev Tooling

  • Git (only bash, avoid any graphic client)
  • Nvm (installer for Nodejs): curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.1/install.sh | bash
  • Install yarn: sudo npm install -g yarn
  • VSCode: https://code.visualstudio.com/download
  • VSCode Extensions:
    • Code Spell Checker
    • Prettier
    • ESLint
    • ErrorLens
  • Extended AWS Role (Chrome Extension): https://chrome.google.com/webstore/detail/aws-extend-switch-roles/jpmkfafbacpgapdghgdpembnojdlgkdl
  • Slack: https://slack.com/intl/es-ar/downloads
  • Postman: https://www.postman.com/downloads/

Although we use several npm packages as TS, CDK and ts-node in every project, we recommend DO NOT install them globally in order to maintain versions compatibility between projects.

Required Tooling Configs

  • GSuite: Just login with your given credentials.
  • AWS
    • Web Console:
      • Go to aws.sinapsis.co and then just login with your given credentials. This is our root account and the entry point for all our accounts and is needed for the next step.
      • In order to switch between accounts in the web console, you’ll need configure Extended AWS Role extension, pasting the content of console-config.txt (attached in this email) in the extension configuration. (img_1)
      • To finish with the console configuration, the last thing is configure the MFA for your account.
    • Profile:
      • To the get programmatic access via aws-cli and aws-sdk locally you’ll need to download aws-config.txt, replace in the aws_access_key_id and aws_secret_access_key (it’s two replacements each) and then paste the content in ~/.aws/credentials (cat ~/Downloads/config.txt >> ~/.aws/credentials). IMPORTANT: If you already have content in credentials, please take care of default profile overlapping.
  • GitHub: Join via email invite.
  • Postman: Join via email invite.

Cloud Components Registry Login

  1. Generate your personal token: https://github.com/settings/tokens/new
    • Permissions: repo (complete) and packages (complete)
    • Save your token safely
  2. Login
    npm login --registry=https://npm.pkg.github.com
    • In the password field use your personal token
